# Paperclip 项目说明

## 什么是 Paperclip？

**Paperclip** 是一个开源的 AI 代理编排平台，用于运行"零人工干预的公司"。

如果 OpenClaw 是一个"员工"，那么 Paperclip 就是"公司"本身。

Paperclip 是一个 Node.js 服务器和 React UI，用于编排一个 AI 代理团队来运行一个业务。你可以导入自己的代理、分配目标、从一个仪表板跟踪代理的工作和成本。

它看起来像一个任务管理器，但在底层它具有组织结构图、预算、治理、目标对齐和代理协调功能。

**管理业务目标，而不是拉取请求。**

## 核心流程

| 步骤 | 说明 | 例子 |
|------|------|------|
| **01** | 定义目标 | _"构建第一名 AI 笔记应用，达到 $1M MRR。"_ |
| **02** | 聘请团队 | CEO、CTO、工程师、设计师、营销人员——任何机器人、任何提供商。 |
| **03** | 审批并运行 | 审查策略。设置预算。开始运行。从仪表板监控。 |

## 主要特性

| 特性 | 描述 |
|------|------|
| 🔌 **自带代理** | 任何代理、任何运行时、一个组织结构图。如果它能接收心跳，就被聘用。 |
| 🎯 **目标对齐** | 每个任务都追溯到公司使命。代理知道做什么和为什么。 |
| 💓 **心跳** | 代理按计划唤醒、检查工作并采取行动。委派在组织结构图上下流动。 |
| 💰 **成本控制** | 每个代理有月度预算。当达到限制时，他们停止。没有失控的成本。 |
| 🏢 **多公司** | 一个部署，多个公司。完全数据隔离。一个控制平面管理你的投资组合。 |
| 🎫 **票证系统** | 每次对话都被追踪。每个决定都有说明。完整的工具调用追踪和不可变审计日志。 |
| 🛡️ **治理** | 你是董事会。批准聘用、覆盖策略、暂停或终止任何代理——随时随地。 |
| 📊 **组织结构图** | 层级结构、角色、汇报线。你的代理有老板、职位和工作描述。 |
| 📱 **移动就绪** | 从任何地方监控和管理你的自动化业务。 |

## Paperclip 适合你，如果：

- ✅ 你想构建**自主 AI 公司**
- ✅ 你**协调许多不同的代理**（OpenClaw、Codex、Claude、Cursor）实现共同目标
- ✅ 你有**20个同时打开的 Claude Code 终端**，不知道谁在做什么
- ✅ 你想要代理**自主运行 24/7**，但仍想审计工作并在需要时介入
- ✅ 你想**监控成本**并执行预算
- ✅ 你想要一个管理代理的流程，感觉像使用任务管理器
- ✅ 你想从手机上**管理你的自动化业务**

## 系统架构

Paperclip 是一个完整的控制平面，包含以下系统：

### 核心系统

1. **身份和访问** - 两种部署模式（受信任本地或已认证）、董事会用户、代理 API 密钥、短期运行 JWT、公司成员资格、邀请流程、OpenClaw 上线流程。每个修改请求都追踪到一个执行者。

2. **组织结构图和代理** - 代理有角色、职位、汇报线、权限和预算。适配器示例包括：Claude Code、Codex、CLI 代理（如 Cursor/Gemini/bash）、HTTP/webhook 机器人（如 OpenClaw）和外部适配器插件。

3. **工作和任务系统** - 问题携带公司/项目/目标/父级链接、原子性检出与执行锁、第一类阻塞程序依赖、评论、文件、附件、工作产品、标签和收件箱状态。没有重复工作，没有丢失上下文。

4. **心跳执行** - DB 支持的唤醒队列，带有合并、预算检查、工作区解析、秘密注入、技能加载和适配器调用。运行产生结构化日志、成本事件、会话状态和审计跟踪。恢复自动处理孤立的运行。

5. **工作区和运行时** - 项目工作区、隔离执行工作区（git worktrees、操作员分支）和运行时服务（开发服务器、预览 URL）。代理在正确的目录中使用正确的上下文工作。

6. **治理和批准** - 董事会批准工作流、带有审查/批准阶段的执行策略、决策追踪、预算硬停、代理暂停/恢复/终止和完整审计日志。

7. **预算和成本控制** - 按公司、代理、项目、目标、问题、提供商和模型跟踪代币和成本。范围预算策略，带有警告阈值和硬停。超支暂停代理并自动取消排队的工作。

8. **例程和计划** - 带有 cron、webhook 和 API 触发的重复任务。并发和追赶策略。每个例程执行创建一个追踪问题并唤醒分配的代理——无需手动启动。

9. **插件** - 实例范围的插件系统，带有进程外工作程序、功能门控主机服务、作业调度、工具公开和 UI 贡献。扩展 Paperclip 而无需分叉。

10. **秘密和存储** - 实例和公司秘密、加密本地存储、提供商支持的对象存储、附件和工作产品。敏感值保持在提示之外，除非范围运行明确需要它们。

11. **活动和事件** - 修改操作、心跳状态更改、成本事件、批准、评论和工作产品被记录为可持续活动，以便操作者可以审计发生了什么以及为什么。

12. **公司可移植性** - 导出和导入整个组织——代理、技能、项目、例程和问题——带有秘密清理和碰撞处理。一个部署，多个公司，完全数据隔离。

## Paperclip 不是

| 特性 | 说明 |
|------|------|
| **不是聊天机器人** | 代理有工作，不是聊天窗口。 |
| **不是代理框架** | 我们不告诉你如何构建代理。我们告诉你如何运行一个由它们组成的公司。 |
| **不是工作流构建器** | 没有拖放管道。Paperclip 模型化公司——具有组织结构图、目标、预算和治理。 |
| **不是提示管理器** | 代理带来自己的提示、模型和运行时。Paperclip 管理它们工作的组织。 |
| **不是单代理工具** | 这是为团队设计的。如果你有一个代理，你可能不需要 Paperclip。如果你有二十个——你肯定需要。 |
| **不是代码审查工具** | Paperclip 编排工作，而不是拉取请求。带上你自己的审查流程。 |

## 快速开始

开源。自托管。无需 Paperclip 账户。

```bash
npx paperclipai onboard --yes
```

或者手动：

```bash
git clone https://github.com/paperclipai/paperclip.git
cd paperclip
pnpm install
pnpm dev
```

这在 `http://localhost:3100` 启动 API 服务器。嵌入式 PostgreSQL 数据库自动创建——无需设置。

**要求：** Node.js 20+，pnpm 9.15+

## 开发命令

```bash
pnpm dev              # 完整开发（API + UI，监视模式）
pnpm dev:once         # 完整开发无文件监视
pnpm dev:server       # 仅服务器
pnpm build            # 构建所有
pnpm typecheck        # 类型检查
pnpm test             # 便宜的默认测试运行（Vitest 仅）
pnpm test:watch       # Vitest 监视模式
pnpm test:e2e         # Playwright 浏览器套件
pnpm db:generate      # 生成数据库迁移
pnpm db:migrate       # 应用迁移
```
